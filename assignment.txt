Focus on anything a real user would experience, and document reproducible defects with evidence.

Primary goals (what to find)
- Functionality bugs in UI flows (profile, resume generation, navigation, saving).
- API validation and client-side error handling issues.
- Security vulnerabilities visible at the frontend level (exposed PII, tokens in storage, XSS vectors).
- Functionality gaps (missing states, missing validation, confusing UX).
- Misconfigurations (broken links, wrong endpoints, environment flags).
- Performance and UX optimizations (slow pages, heavy bundles).
- AI agent UX issues (unclear progress states, unhelpful errors).
- LaTeX compile UX surface issues (preview hangs, download errors).
- UI/UX optimizations (accessibility, responsiveness, consistency).
- Session management problems (session expiry, refresh, concurrent logins).
- File handling issues (profile image upload, size limits, error handling).
- Feature suggestions based on observed flows.

Detailed test checklist & actions
1. Core flows
   - Login / logout / session persistence / refresh.
   - Full profile create/edit/save for all sections (use sample data).
   - Resume generation: submit JD, poll status, view source, download PDF.
   - PDF preview rendering (desktop & mobile).
2. Error handling & API handling
   - Cause backend validation errors and confirm mapped UI messages display correctly.
   - Verify UI behavior on HTTP 4xx, 5xx, timeouts, and network interruption.
   - Test idempotency: re-submit resume requests with same Idempotency-Key and verify single generation.
3. Session & auth
   - Verify session cookie attributes in dev/production (HttpOnly, Secure flags where applicable).
   - Test session expiry behavior: expired session should redirect to login without exposing data.
   - Test concurrent sessions (login on another device) and UI behavior.
4. File handling
   - Profile picture upload: validate file types, sizes, thumbnails, broken upload recovery, large image behavior.
   - Resume download file integrity (non-zero PDF, correct headers, content-disposition).
5. UI/UX & accessibility
   - Keyboard navigation for side menu, profile dropdown; aria attributes present; role=alert for error banners.
   - Contrast and readable text in dark theme.
   - Responsive layout checks at desktop/tablet/mobile.
   - Highlighter behavior in side menu (active state follows scroll).
6. Performance & optimization
   - Run Lighthouse (desktop & mobile) and record scores; flag >5s interactive or large bundle assets.
   - Test cold load and repeat navigation performance.
7. AI & LaTeX UX
   - Simulate slow/failed AI responses; verify clear messages and no silent failures.
   - Validate retry UI behavior and that compile logs (when surfaced) are user-safe.
8. Security surface checks (non-destructive)
   - Confirm no tokens/PII in localStorage/sessionStorage or visible in console.
   - Test for obvious XSS vectors (enter script-like input into visible text fields and confirm escaped rendering).
   - Verify CORS behavior, mixed-content warnings, and secure contexts.
9. Miscellaneous
   - Verify all links, help texts, and tooltips are accurate.
   - Suggest small UX improvements to reduce friction.

Tools & techniques (recommended)
- Chrome/Firefox devtools, network throttling
- Lighthouse, WebPageTest if available
- Accessibility tools (axe, Lighthouse a11y)
- Playwright or Cypress for reproducible E2E (optional)
